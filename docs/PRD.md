# 照片归档备份工具 PRD（v1.0）

## 1. 产品概述

### 1.1 背景
相机和手机拍摄素材常通过内存卡导出，手工整理存在重复导入、命名冲突、备份不完整、后续检索困难等问题。

### 1.2 产品目标
- 将内存卡素材安全、可追踪地归档到备份目录。
- 通过 GUI 降低使用门槛，提供可视化任务管理。
- 通过图片内容识别实现标签化检索。

### 1.3 非目标（当前版本）
- 不做云相册托管与跨设备实时同步。
- 不做在线多人协同编辑。

## 2. 用户与场景

### 2.1 目标用户
- 摄影爱好者
- 内容创作者
- 家庭照片管理用户

### 2.2 核心场景
- 插入 SD 卡后执行导入，按日期自动归档。
- 重复导入时自动跳过重复文件。
- 导入完成查看报告并重试失败项。
- 在图库按标签搜索（如：人物、海边、日落）。

## 3. 需求范围

### 3.1 Phase 1（MVP）
- 扫描源目录（内存卡挂载路径或手动选择）
- 白名单文件筛选
- 时间提取（EXIF 优先，mtime 回退）
- 目标路径规划（按模板）
- 复制、冲突处理、校验、去重
- GUI：导入页、任务记录页、设置页
- 导入报告与失败重试

### 3.2 Phase 2（增强）
- SQLite 增量索引与断点续跑
- 双目标备份
- RAW+JPEG 配对归档
- 并发与 I/O 限速
- 图库页、标签搜索
- 图片内容识别（异步任务）
- 标签人工修正
- macOS 原生 SwiftUI 前端（通过本地 API 对接 Go Core）

## 4. 功能需求

### 4.1 导入任务
- 输入：源目录、目标目录、可选副备目录。
- 支持格式：jpg/jpeg/heic/png/cr2/cr3/nef/arw/dng/mp4/mov。
- 目录模板：默认 `{YYYY}/{YYYY-MM-DD}`。
- 重名策略：保留原名，冲突自动追加序号 `_001`。
- 去重策略：`hybrid`（快速判重 + hash 确认）默认。
- 校验策略：`size` 默认，`hash` 可选。
- 删除源文件：默认关闭，仅在显式开启时允许。

### 4.2 GUI
- 导入页：来源/目标选择、预扫描、开始、取消、进度。
- 任务记录页：任务状态、统计、失败列表、重试。
- 设置页：模板、并发数、去重/校验模式、识别参数。
- 图库页（Phase 2）：缩略图浏览、筛选和搜索。

### 4.3 图片识别（Phase 2）
- 导入后异步执行，不阻塞导入主流程。
- 输出标签与置信度，低于阈值不入库。
- 标签来源区分：`ai` 与 `manual`。
- 支持人工添加/删除标签。

## 5. 非功能需求

- 稳定性：单文件失败不影响全任务。
- 可恢复：任务中断后可重跑。
- 性能：支持万级文件批量导入。
- 可观测：日志、错误码、任务报告可回溯。
- 安全：默认不删除源文件，不上传用户数据。

## 6. 验收标准（DoD）

- 成功导入并归档混合格式文件（图片/RAW/视频）。
- 重复导入时重复文件自动跳过。
- 校验失败文件必须标记失败，不可误报成功。
- GUI 能完整执行一次导入并展示结果统计。
- 失败项可重试，且重试结果可追踪。
- Phase 2：标签搜索可返回正确文件集合。

## 7. 里程碑建议

- M1：导入核心、CLI、SQLite 基础
- M2：GUI 导入页 + 任务记录页
- M3：双备份、性能优化、失败重试
- M4：识别能力、图库检索、标签编辑
